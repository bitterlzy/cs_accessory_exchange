> 版本: v2.1 | 日期: 2026-06-06

## 如何阅读

每个测试用例包含:
| 要素 | 说明 |
|---|---|
| **TC-ID** | 测试用例唯一编号 |
| **标题** | 测试什么 |
| **优先级** | P0(必须过)/P1(重要)/P2(增强) |
| **前置条件** | 运行前需要满足的条件 |
| **测试步骤** | 按顺序执行的操作 |
| **预期结果** | 每一步期望的输出 |

## 10. 集成测试: 完整交换流程

| 步骤 | 操作 | 预期 |
|---|---|---|
| 1 | Alice 注册 (POST /auth/register) | 201, 获得 token |
| 2 | Bob 注册 (POST /auth/register, 不同手机号) | 201, 获得 token |
| 3 | Alice 添加饰品 (POST /inventory) | 201, 获得 item_id |
| 4 | Alice 发布请求 (POST /listings) | 201, 获得 listing_id |
| 5 | Bob 添加饰品 (POST /inventory) | 201 |
| 6 | Bob 发起提议 (POST /offers) | 201, 获得 offer_id |
| 7 | Alice 查看通知 (GET /notifications) | 包含新提议通知 |
| 8 | Alice 接受提议 (POST /offers/{id}/accept) | 200, status=accepted |
| 9 | 双方在 Steam 完成交易 (外部操作) | — |
| 10 | Bob 提交 Steam 报价 (POST /steam/offers/{id}/submit) | 200 |
| 11 | Alice 确认到账 (POST /steam/offers/{id}/confirm) | 200, 交换完成 |
| 12 | 验证库存 (GET /inventory) | 物品已交换 |
| 13 | 查看交换历史 (GET /trades) | 包含本次记录 |

## 11. 安全相关验证

| TC-ID | 验证点 | 验证方法 |
|---|---|---|
| TC-SEC-001 | 一人一号 - 身份证查重 | 同一身份证号注册两个账号 → 第二个被拒绝 |
| TC-SEC-002 | 一人一号 - 手机号查重 | 同一手机号注册两个账号 → 第二个被拒绝 |
| TC-SEC-003 | 一人一号 - 支付宝查重 | 同一支付宝绑定两个账号 → 第二个被拒绝 |
| TC-SEC-004 | Steam 终生绑定 | 绑定后尝试解绑或换绑 → 接口不存在或被拒绝 |
| TC-SEC-005 | Steam 唯一绑定 | 两个用户绑定同一 Steam → 第二个被拒绝 |
| TC-SEC-006 | 物品互斥锁 | 同一物品同时上架和交易 → 第二个操作被拒绝 |
| TC-SEC-007 | 所有权验证 | 操作他人物品 → 403 拒绝 |
| TC-SEC-008 | JWT 过期 | 使用过期 token → 401 拒绝 |

## 12. 测试环境准备

```bash
# 1. 启动 MySQL + 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS cs_trade_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. 安装依赖
pip install -r requirements.txt

# 3. 初始化数据
python seed.py

# 4. 启动服务器
uvicorn app.main:app --host 0.0.0.0 --port 3000 --reload

# 5. 打开 API 文档
# Swagger: http://localhost:3000/docs
```
